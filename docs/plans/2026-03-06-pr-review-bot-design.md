# PR Review Bot - Auto Code Review on PR Open

**Date:** 2026-03-06
**Status:** Approved

## Problem

When a PR is opened on `TheLukasHenry/proxy-server`, the webhook-handler forwards it to n8n for review — but n8n's API key is broken (401). No automatic review happens.

## Solution

Bypass n8n entirely. Use the existing Open WebUI AI + GitHub API clients already in webhook-handler to perform the review directly.

## Flow

```
GitHub PR opened/synchronize
  -> webhook-handler receives event
  -> Fetch PR details + file list via GitHubClient
  -> Send to OpenWebUIClient.analyze_pull_request()
  -> Post review as GitHub comment (GitHubClient.post_issue_comment)
  -> Post summary to Discord (_notify_discord)
```

## Changes

### File: `webhook-handler/handlers/github.py`

Modify `_handle_pull_request_event`:
- On `opened` or `synchronize` actions, instead of forwarding to n8n:
  1. Parse owner/repo from payload
  2. Call `self.github.get_pr_details(owner, repo, pr_number)` + `self.github.get_pr_files(owner, repo, pr_number)`
  3. Call `self.openwebui.analyze_pull_request(title, body, diff_summary, labels)`
  4. If AI returns a review, post it via `self.github.post_issue_comment(owner, repo, pr_number, formatted_review)`
  5. Post a short Discord summary via `self._notify_discord()`
- Keep n8n forwarding as optional fallback (if n8n is configured and working)

### No new files needed

All clients and methods already exist:
- `GitHubClient.get_pr_details()` - fetches PR metadata + files
- `GitHubClient.get_pr_files()` - fetches file change summary
- `GitHubClient.post_issue_comment()` - posts comment on PR
- `OpenWebUIClient.analyze_pull_request()` - AI review
- `GitHubClient.format_ai_response()` - formats as markdown comment

## Review Level

File-level review: AI sees which files changed, line counts, PR title/body. Does not fetch actual code diffs.

## Output Format

### GitHub Comment
```
AI Analysis

Summary: [what changed]
Potential Issues: [concerns]
Suggestions: [improvements]
Overall Assessment: [verdict]

---
Generated by Open WebUI AI Assistant
```

### Discord Notification
```
AI Review for PR #N: title
by author -> base_branch
[1-2 sentence summary]
link
```

## Error Handling

- If Open WebUI is down (401/timeout), skip AI review, still send Discord PR notification (already working)
- If GitHub comment post fails, log error but don't fail the webhook response
- If Discord notification fails, log warning but don't fail
